# Anime Pipeline 使用文档

## 一、项目概述

Anime Pipeline 是一个**动漫资源下载、字幕提取、视频切割、音频降噪、ASR 语音识别**的一体化工具链。核心工作流：搜索动漫资源 → 下载 → 提取中文字幕 → 按字幕时间轴切割视频片段 → 人工审核筛选 → 提取音频 → AI 降噪 → 音频质量过滤，最终产出可用于语音训练的高质量日语音频片段。

技术栈：**Python + FastAPI 后端 + 单文件 HTML 前端**，本地运行在 `http://localhost:5800`。

## 三、快速开始

### 新环境一键配置

**Windows:**
```bash
双击运行 setup.bat
```

**Linux/macOS:**
```bash
chmod +x setup.sh && ./setup.sh
```

脚本会自动完成：
1. 检测 Python 环境
2. 创建虚拟环境并安装依赖
3. **检测并清理旧机器的 settings.json**（自动适配新环境路径）
4. 创建所有数据目录
5. 检查外部工具（ffmpeg、mkvtoolnix）
6. 可选：安装 ClearVoice 降噪引擎
7. 启动服务器

### 自定义数据路径

默认所有数据目录在 `data/` 下。如需修改（例如移动到其他硬盘），有两种方式：

**方式一：Web UI**
启动后在 `http://localhost:5800` → 设置 → 文件路径 → 修改并保存。

**方式二：直接编辑 `data/settings.json`**
```json
{
  "paths": {
    "CLIPS_DIR": "D:/MyAnime/clips",
    "DOWNLOAD_DIR": "D:/MyAnime/downloads"
  }
}
```

只支持修改 `_USER_PATH_KEYS` 中列出的路径（见 config.py）。
非路径设置（模型选择、步骤顺序等）即使 settings.json 被清除也会保留。

### 日常快速启动

```bash
# Windows
双击 start.bat

# Linux/macOS
./start.sh
```

### 离线环境部署

如果目标服务器没有网络，先在联网机器上准备离线包：

```bash
# 下载所有 wheel 包
python tools/download_wheels.py --linux

# 将 offline_wheels/ 目录复制到目标服务器
# 然后在服务器上运行
./setup.sh
```

## 四、核心工作流

```
搜索资源 → 提交下载(aria2c后台) → 文件监视器检测完成
    → 提取中文字幕(track) → 按字幕时间轴切割视频片段
    → 过滤过短视频(<1秒) → 人工审核(试听/批准/跳过)
    → MP4→WAV 转换 → AI降噪(ClearVoice 多步骤)
    → 音频质量筛选(混响/静音/VAD/PAD) → 最终输出
```

## 五、各模块功能详解

### 5.1 资源搜索（downloader.py）
- 对接 **animes.garden** API，聚合 dmhy / moe / ani 三个资源站
- 支持关键字搜索、字幕组过滤、资源类型过滤、分页
- 返回磁力链接、文件大小、格式（MKV/MP4）、字幕组等信息

### 5.2 下载管理（aria2_rpc.py / qbittorrent / bitcomet）
- **aria2c**（默认）：启动后台守护进程（端口 6800），JSON-RPC 通信
- **qBittorrent**：启动 GUI 客户端添加磁力链接
- **BitComet**：启动 GUI 客户端并配置下载目录
- 下载完成后文件监视器自动触发后续处理

### 5.3 文件监视器（file_watcher.py）
- 后台每 5 秒扫描 `downloads/` 和 `video/` 目录
- 检测新 MKV/MP4 文件后自动触发提取+切割流水线
- 通过文件大小稳定性判断（2秒内不变）避免处理未完成文件

### 5.4 字幕提取（extract_subs.py）
- 使用 mkvmerge -J 获取 MKV 内所有轨道信息
- 智能识别中文字幕轨道（支持 chi/zho/zh-CN/zh-Hans 等编码）
- 优先简体中文（zh-Hans），跳过繁体中文（zh-Hant）
- 使用 mkvextract（首选）或 ffmpeg（备用）提取

### 5.5 视频切割（split_video.py）
- 三种切割模式：字幕时间轴 / 等时长 / 等文件大小
- 支持硬件加速编码：NVENC（NVIDIA）/ AMF（AMD）/ QSV（Intel）/ libx264（软件）
- 字幕重叠去重（deduplicate_entries）
- 硬编码失败时自动回退到 libx264 软件编码
- 并发控制：同一时间只允许一个切割任务（GPU 资源保护）

### 5.6 AI 降噪（denoise_audio.py）
动态可选择和排序的降噪步骤：
| 步骤 | 功能 |
|------|------|
| music_separate | BGM 分离（Demucs HT） |
| clear_short | 清除 ≤2s 的过短音频 |
| male_voice | 去除男声（ONNX Wav2Vec2 / YIN 音高） |
| enhance | 语音增强（MossFormer2_SE_48K） |
| super_resolve | 超分辨率（MossFormer2_SR_48K） |
| reverb | 混响检测 |
| silence | 静音比例过滤 |
| vad | VAD 噪声事件检测 |
| pad | 首尾静音规范化（0.5s） |

### 5.7 ASR 语音识别（asr_pipeline.py）
支持多模型：
| 模型 | 框架 | 特点 |
|------|------|------|
| Qwen3-ASR-1.7B | qwen-asr | 阿⾥多语言，高精度 |
| SenseVoiceSmall | FunASR | 阿⾥多语言，轻量快速 |
| Paraformer-Large | FunASR | 中文特化，极快推理 |
| Whisper Large v3 | Transformers | OpenAI，99种语言 |
| Cohere Transcribe | Transformers | 14种语言含日语 |
| Parakeet-TDT-0.6B-ja | NeMo | NVIDIA 日语特化 |
| FireRedASR2-AED | FireRedASR2S | 工业级全功能（VAD+语种+标点） |

## 七、配置说明

### 外部工具依赖
| 工具 | 位置 | 用途 |
|------|------|------|
| ffmpeg / ffprobe | `ComicCut/QuickCut/` | 视频编解码、音频转换 |
| mkvextract / mkvmerge / mkvinfo | `ComicCut/mkvtoolnix/` | MKV 轨道提取与分析 |
| aria2c | 系统路径或 `tools/` | P2P 下载 |
| ClearVoice | `ComicCut/ClearerVoice-Studio-main/` | AI 语音增强/超分 |

### Python 依赖
- **核心**：`fastapi`, `uvicorn`, `requests`, `librosa`, `torch`, `torchaudio`, `demucs`
- **ASR（可选）**：`funasr`, `transformers`, `nemo_toolkit`, `qwen-asr`, `onnxruntime`

### 自定义配置
- `data/settings.json` — 路径覆盖、降噪步骤、ASR 模型选择
- `config.py` — 全局默认值（`_CONFIG_KEYS`、`_USER_PATH_KEYS`）

## 八、关键设计决策

1. **并发控制**：视频切割使用信号量限制为 1（GPU 资源保护），流水线任务限制为 3 个并发
2. **可取消性**：所有耗时操作支持通过 `threading.Event` 信号取消
3. **状态持久化**：任务状态实时写入 `jobs.json`，服务重启后可恢复
4. **文件监视器**：启动时自动将已有文件标记为已知（跳过），避免重复处理
5. **硬编码回退**：GPU 编码失败自动回退到 CPU（libx264 ultrafast）
6. **模型懒加载**：AI 模型按需加载，单例模式避免重复加载
7. **跨平台兼容**：setup.sh 自动处理网络挂载、无网络离线安装等边缘情况
8. **新环境自适应**：启动时自动检测 settings.json 是否为其他机器的陈旧配置，自动重置路径
