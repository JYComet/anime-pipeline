# Anime Pipeline 使用文档

## 一、项目概述

Anime Pipeline 是一个**动漫资源下载、字幕提取、视频切割、音频降噪**的一体化工具链。核心工作流：搜索动漫资源 → 下载 → 提取中文字幕 → 按字幕时间轴切割视频片段 → 人工审核筛选 → 提取音频 → AI 降噪 → 音频质量过滤，最终产出可用于语音训练的高质量日语音频片段。

技术栈：**Python + FastAPI 后端 + 单文件 HTML 前端**，本地运行在 `http://localhost:5800`。

## 三、核心工作流

```
搜索资源 → 提交下载(aria2c后台) → 文件监视器检测完成
    → 提取中文字幕(track) → 按字幕时间轴切割视频片段
    → 过滤过短视频(<1秒) → 人工审核(试听/批准/跳过)
    → MP4→WAV 转换 → AI降噪(ClearVoice 3阶段)
    → 音频质量筛选(混响/静音/VAD/PAD) → 最终输出
```

## 四、各模块功能详解

### 4.1 资源搜索（downloader.py）

- 对接 **animes.garden** API，聚合 dmhy / moe / ani 三个资源站
- 支持关键字搜索、字幕组过滤、资源类型过滤、分页
- 返回磁力链接、文件大小、格式（MKV/MP4）、字幕组等信息

### 4.2 下载管理（aria2_rpc.py）

- 启动 aria2c 守护进程（端口 6800），通过 JSON-RPC 通信
- 支持 DHT 网络、Peer Exchange、多 Tracker
- 后台下载不阻塞 UI，下载完成后文件监视器自动触发后续处理
- 同时提供 magnet 链接的解析和文件列表预览

### 4.3 文件监视器（file_watcher.py）

- 后台每 5 秒扫描 `downloads/` 和 `video/` 目录
- 检测新 MKV 文件后自动触发提取+切割流水线
- 通过文件大小稳定性判断（2秒内不变）避免处理未完成文件
- 跳过 .aria2 临时文件

### 4.4 字幕提取（extract_subs.py）

- 使用 mkvmerge -J 获取 MKV 内所有轨道信息
- 智能识别中文字幕轨道（支持 chi/zho/zh-CN/zh-Hans 等编码）
- 优先简体中文（zh-Hans），跳过繁体中文（zh-Hant）
- 使用 ffmpeg（首选）或 mkvextract（备用）提取
- 支持 ASS/SSA/SRT 格式

### 4.5 视频切割（split_video.py）

- 解析 ASS/SRT 字幕文件获取每条字幕的起止时间
- 使用 ffmpeg 按时间轴将视频切成独立片段（mp4）
- 支持硬件加速编码：NVENC（NVIDIA）/ AMF（AMD）/ QSV（Intel）/ libx264（软件）
- 自动检测可用的硬件加速方案
- 支持取消操作（cancel_event）
- 并发控制：同一时间只允许一个切割任务（GPU 资源保护）
- 硬编码失败时自动回退到 libx264 软件编码
- 文件名格式：`{视频名}_S{序号}_{字幕文本前20字}.mp4`

### 4.6 视频审核（server.py review endpoints）

- 提供视频片段试听/审核界面
- 批准：将 MP4 移动到 approved/ 目录，自动转换为 WAV 音频
- 跳过：标记为跳过，下次不再出现
- 审核状态持久化到 `.review_state.json`

### 4.7 音频转换（convert_audio.py）

- ffmpeg 提取音轨：16-bit PCM WAV, 44100Hz, 立体声
- 去除视频流，仅保留音频
- 支持批量转换

### 4.8 AI 降噪（denoise_audio.py）

三阶段 ClearVoice 处理：

| 阶段 | 模型 | 功能 |
|------|------|------|
| 1a | MossFormer2_SE_48K | 语音增强（去噪） |
| 1b | MossFormer2_SS_16K | 语音分离 + 多说话人检测 |
| 1c | MossFormer2_SR_48K | 语音超分辨率（16kHz→48kHz 带宽扩展） |

- 多说话人检测：若分离出的两条音轨 RMS 能量比 > 0.3，判定为多人声→丢弃
- 模型懒加载（单例模式），节省内存

### 4.9 音频质量筛选（audio_pipeline.py）

四步骤过滤（在降噪后执行）：

| 步骤 | 功能 | 丢弃条件 |
|------|------|----------|
| 混响检测 | 检测尾部能量比和零交叉率 | energy_ratio > 0.08 或 ZCR > 3000Hz |
| 静音比例 | 统计 RMS < 0.006 的帧占比 | 静音 > 50% |
| VAD 检测 | 语音活动检测，找噪声事件 | 长非语音段（>1s）且能量 > 0.001 |
| PAD 规范 | 将首尾静音统一为 0.5s | — |

## 七、使用流程

### 7.1 在线下载模式
1. 在「搜索资源」页输入动漫名称（支持日文/中文/罗马音）
2. 可选过滤字幕组、资源类型
3. 点击资源的「下载」按钮，后台 aria2c 开始 P2P 下载
4. 下载完成后系统自动提取字幕并切割视频
5. 进入「结果」页查看片段

### 7.2 本地文件模式
1. 将 MKV/MP4 文件放入 `video/` 目录（或 `downloads/` 目录）
2. 文件监视器自动检测并触发处理
3. 或在「本地文件」页手动选择视频 + 外挂字幕处理
4. MP4 文件（无内置字幕轨）需要手动提供外挂 ASS/SRT 字幕

### 7.3 审核筛选
1. 进入前端「审核」页面
2. 逐个试听/观看视频片段
3. 需要的片段点「通过」（自动转 WAV）
4. 不需要的点「跳过」
5. 审核状态自动保存

### 7.4 音频降噪
1. 在「降噪」页选择已批准的 WAV 文件
2. 点击「批量降噪」
3. 系统自动执行 ClearVoice 3 阶段降噪 + 4 步质量过滤
4. 最终输出在 `data/cleaned/{视频名}/` 目录下，文件以 `_norm.wav` 结尾

## 八、配置说明（config.py）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `ANIMEGARDEN_API` | `api.animes.garden` | 资源搜索 API |

| 混响阈值 | `TAIL_ENERGY_RATIO_THRESH = 0.08` | 高于此值判定有混响 |
| 静音阈值 | `MAX_SILENCE_RATIO = 0.5` | 静音超过 50% 丢弃 |
| 多人声阈值 | `MULTI_SPEAKER_RMS_RATIO = 0.3` | 第二音轨能量 > 主音轨 30% 则为多人声 |

## 九、数据流与文件命名规范

### 文件转换链
```
[原始]  {视频名}.mkv
  → 字幕:  {视频名}_track{N}_{lang}.ass      (subtitles/)
  → 片段:  {视频名}_S{序号}_{字幕文本}.mp4    (clips/{视频名}/)
  → 批准:  同上 .mp4 + .wav                   (approved/{视频名}/)
  → 增强:  {原名}_enhanced.wav                (cleaned/{视频名}/)
  → 分离:  {原名}_enhanced_s1.wav / _s2.wav   (cleaned/{视频名}/separated/)
  → 超分:  {原名}_sr.wav                      (cleaned/{视频名}/)
  → 最终:  {原名}_norm.wav                    (cleaned/{视频名}/)
```

### 目录映射
```
data/clips/{视频名}/          → 切割后所有片段
data/approved/{视频名}/       → 审核通过的片段 + WAV
data/cleaned/{视频名}/        → 降噪完成后的最终音频
```

## 十一、关键设计决策

1. **并发控制**：视频切割使用信号量限制为 1（GPU 资源保护），流水线任务限制为 3 个并发
2. **可取消性**：所有耗时操作（下载、提取、切割）支持通过 `threading.Event` 信号取消
3. **状态持久化**：任务状态实时写入 `jobs.json`，服务重启后可恢复
4. **文件监视器**：启动时自动将正在进行中的下载标记为已知（跳过），避免重复处理
5. **硬编码回退**：GPU 编码失败自动回退到 CPU（libx264 ultrafast）
6. **模型懒加载**：AI 模型按需加载，单例模式避免重复加载
